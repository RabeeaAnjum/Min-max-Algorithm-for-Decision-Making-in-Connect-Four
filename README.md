# Min-max-Algorithm-for-Decision-Making-in-Connect-Four

# 🎮 Connect Four AI using Minimax Algorithm

An Artificial Intelligence implementation of the classic **Connect Four** game using the **Minimax Algorithm** for optimal decision-making. The computer acts as an intelligent player that evaluates all possible moves and chooses the best one, assuming both players play optimally.

---

# 📌 Project Overview

Connect Four is a two-player strategy game played on a **6 × 7** grid. Players take turns dropping discs into one of the seven columns. The first player to align **four consecutive discs** horizontally, vertically, or diagonally wins the game.

This project demonstrates how the **Minimax Algorithm** can be applied to game-playing AI by recursively exploring possible game states and selecting the optimal move.

# 🎮 How to Play

When the game starts, an empty **6 × 7** board is displayed.

```
 | | | | | |
-------------
 | | | | | |
-------------
 | | | | | |
-------------
 | | | | | |
-------------
 | | | | | |
-------------
 | | | | | |
-------------
```

The human player uses **O**.

The computer uses **X**.

Enter a column number between **1 and 7** to drop your piece.

Example:

```
Enter your move (1-7): 4
```

The computer automatically calculates the best move using the Minimax algorithm.

---

# 🧠 Minimax Algorithm

The AI follows these steps:

1. Generate all legal moves.
2. Simulate every possible move.
3. Evaluate resulting game states.
4. Assume both players make optimal decisions.
5. Choose the move with the highest utility.

### Utility Function

| Game State | Score |
|------------|------:|
| Computer Wins | +1 |
| Human Wins | -1 |
| Draw | 0 |

# 📚 Functions Used

Checks for:

- Diagonal win

Returns:

# 📈 Time Complexity

The Minimax algorithm explores the complete game tree.

- **Worst Case:** Exponential
- **Time Complexity:** **O(bᵈ)**

Where:

- **b** = branching factor
- **d** = search depth

Since Connect Four has a very large search space, plain Minimax becomes computationally expensive.
